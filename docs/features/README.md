## Feature Template

Every feature document follows this structure:

1. **Purpose** — Why this feature exists
2. **Users and Roles** — Who uses it
3. **Business Value** — What problem it solves
4. **User Flow** — Mermaid diagram of user journey
5. **Business Rules** — Rules governing behavior
6. **System Behavior** — What the system does
7. **Data and Configuration** — Tables, fields, settings
8. **Permissions and Security** — RBAC matrix
9. **Integrations** — External system dependencies
10. **Edge Cases** — Boundary conditions
11. **Acceptance Criteria** — Testable requirements
12. **Current Implementation Status** — What's built vs planned
13. **Open Questions** — Unresolved items

> [!NOTE]
> Additional feature specs live in `specs/` directory. See also `docs/superpowers/specs/` for design documents.
