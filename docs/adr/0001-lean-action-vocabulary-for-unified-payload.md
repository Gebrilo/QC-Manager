# Use a lean action vocabulary for unified Tuleap payloads

The unified Tuleap payload carries an `action` field with four values — `sync`, `delete`, `reject`, `archive` — rather than the more conventional `create | update | delete | reject | archive`. We picked the lean enum because n8n cannot reliably know from a Tuleap webhook whether the artifact is new or existing, and both branches end up upserting by `tuleap_artifact_id` anyway, so naming the action `sync` is honest about what the persistence layer actually does. `reject` and `archive` only apply to tasks (unknown assignee at intake; reassignment out of the QC team) — sending either on any non-task `artifact_type` returns HTTP 400 rather than silently no-op'ing, so misrouted webhooks fail loudly instead of quietly dropping work.
